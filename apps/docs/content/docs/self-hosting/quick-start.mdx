Use this path if you want the most direct setup.

## Prerequisites

- Docker Engine 24+ and Docker Compose v2
- 2 vCPU / 4 GB RAM / 40 GB SSD recommended baseline
- Linux VPS (Ubuntu 22.04+ recommended) or local machine with Docker Desktop
- Domain + reverse proxy for production (`80/443`)

Security baseline:

- Expose only `80/443` publicly
- Keep `3000`, `3001`, and `5432` private
- Use a strong `BETTER_AUTH_SECRET`

## 1. Clone the Repository

```bash
git clone https://github.com/redpangilinan/crikket
cd crikket
```

## 2. Create Runtime Env File

```bash
cp .env.example .env
cp apps/server/.env.example apps/server/.env
cp apps/web/.env.example apps/web/.env
```

## 3. Configure Environment

Root `.env` is Docker-only (images/ports/bundled Postgres settings).

App settings belong in:

- `apps/server/.env`
- `apps/web/.env`

For production, set these explicitly in app env files:

```bash
DATABASE_URL
BETTER_AUTH_SECRET (32+ characters)
NEXT_PUBLIC_APP_URL
NEXT_PUBLIC_SERVER_URL
BETTER_AUTH_URL
CORS_ORIGIN
GOOGLE_CLIENT_ID
GOOGLE_CLIENT_SECRET
ENABLE_PAYMENTS (false)
```

## 4. Start Services

Use prebuilt images (default):

```bash
docker compose up -d
```

Build locally from source:

```bash
docker compose -f docker-compose.yml -f docker-compose.build.yml up -d --build
```

## 5. Validate Deployment

```bash
docker compose ps
docker compose logs -f server
docker compose logs -f web
```

Schema migrations run automatically through the `migrate` service before `server` starts.
